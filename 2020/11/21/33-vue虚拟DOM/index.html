为什么要有虚拟DOM?

jQuery/原生JS 自己控制DOM操作
Vue/React 修改数据，数据改变了，视图更新（DOM操作） 数据驱动视图 Vue内部操作DOM
Vue 虚拟DOM
JS计算好变化 -> 操作我们的真实DOM
操作DOM 成本很高 执行JS很快 >>> DOM改变

虚拟DOM -> JS来模拟真实DOM结构

用snabbdom模拟vue的虚拟dom

patch的两个参数放入改变前后的h函数，比较两者的变化，计算出最少的DOM操作，并渲染或更新页面。
